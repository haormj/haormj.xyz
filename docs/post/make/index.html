Why

为什么会出现 make?

出现的初衷是为了编译 c 程序? 还是 c++程序,为了解决当项目太大的时候,繁琐的编辑工作

出现的初衷就是为了解决 c 和 c++编译繁琐的问题

当前也可以解决其他程序编译的问题

为什么编译 go 的时候,都不需要呢?

因为 go build 直接就可以提供一个二进制,根本不需要编译其他的文件

不像 c/c++一样,使用到了其他内容,需要开发者需要编译,链接使用的库

这些工作 go build 直接帮你完成,从而 go 的开发者会轻松很多,不需要管很多内容

从而对于 go 而言,如果自己除了编译还有其他工作要完成

例如编译二进制,拷贝配置文件,生成安装包等内容,是否通过脚本就可以实现呢?

当然可以,执行脚本,然后执行指定的内容,例如编译 例如将二进制上传到制品库,这些都是需要的

这些操作当然可以通过 shell 脚本完成,但是当然也可以通过 make 完成

起码不需要写一堆 case,因为 make 已经帮你做了

可以实现相同的目的,而且让你的脚本更少,而且可以直接打包,不过这个感觉也没什么

但是有一点是可以更加的通用,因为这基本上对于 c 和 c++项目都是必须的

从而对于有基础的人而言应该也没有什么成本,也能够达到目的,这样至少对于打包可执行文件有了统一的做法

对于我而言

  1. 更加通用

  2. 减少项目中的脚本

  3. 目前为了方便有 build.sh pando.sh 但是有了 Makefile 之后就不需要这么多了

  4. 更加熟悉 Makefile/make,让自己对于之前 c/c++的编程不在是那么的排斥

熟悉了这些内容后,基本上就是如何写 Makefile, 以及更加优美的写 Makefile 了

以及自己的最佳实践,这样就足够了
